    VM PART



    - Run a virtual machine with the Linux OS of your choice.

    ⋆ install VirtualBox VM and create a new VM
    ⋆ a disk size of 8 GB
    ⋆ have at least one 4.2 GB partition. For example, 4.6 for /root




    ⋆ be root:
    $ su -
    ⋆ get the info on the newest version of the packages:
    $ apt-get update -y
    ⋆ upgrade to the these newest versions:
    $ apt-get upgrade -y

    NETWORK AND SECURITY PART



    - Create a non-root user to connect to the machine and work. Use sudo, with this user, to be able to perform operation requiring special rights.

    ⋆ install sudo and Vim:
    $ apt-get install sudo vim
    ⋆ an user was created during the VM installation, but if you want to create another one:
    $ adduser [newname]
    ⋆ you will be asked for a new password and some other info for the new user.
    ⋆ then, to add the new user to sudoers group:
    $ usermod -aG sudo [newname]
    ⋆ check who is in the sudo group:
    $ getent group sudo
    ⋆ delete an user:
    $ deluser [username]
    ⋆ switch users:
    $ su - [username]





    - Configure the machine to have a static IP and a Netmask in \30.

    ⋆ you can apt-get install net-tools to check your interface configuration with sudo ifconfig
    ⋆ modify /etc/network/interfaces:
    $ cat /etc/network/interfaces

    # This file describes the network interfaces available on your system
    # and how to activate them. For more information, see interfaces(5).

    source /etc/network/interfaces.d/*

    # The loopback network interface
    auto lo
    iface lo inet loopback


    # The primary network interface
    auto enp0s3
    ⋆ create the following file:
    ⋆ netmask: 255.255.255.252
    ⋆ default gateway: ip r | grep default
    $ cat /etc/network/interfaces.d/enp0s3

    iface enp0s3 inet static
          address [ip]
          netmask [netmask]
          gateway [gateway]

    ⋆ check results: sudo ifconfig
    ⋆ sudo service networking restart





    - Change the default port of the SSH service by the one of your choice.

    $ sudo vim /etc/ssh/sshd_config
    ⋆ change port 22 to a port between 49152 and 65535. Leave uncommented
    $ sudo service sshd restart
    ⋆ test connect to the new port:
    $ ssh -p [port] [user]@[ip]
    ⋆ close the connection:
    $ exit


    - Create an rsa key pair on the host machine.

    ⋆ in /etc/ssh/sshd_config, make sure you have set "#PasswordAuthentication yes" and "PubkeyAutenthication yes"
    ⋆ in VirtualBox manager, your VM → settings → network, change network mode to Bridged Adapter

    ⋆ in host terminal:
    $ ssh-keygen -t rsa
    ⋆ send public key to the virtual machine:
    $ ssh-copy-id -i ~/.ssh/id_rsa.pub [user]@[ip] -p [port]
    ⋆ now you can ssh to VM from host:
    $ ssh -p [port] [user]@[ip]
    $ exit


    - Make SSH root access allowed not directly, but with a user who can be root.

    $ sudo vim /etc/ssh/sshd_config
    ⋆ change the line "#PasswordAuthentication yes" to "passwordAutenthication no"
    $ sudo service sshd restart
    ⋆ now you should be able to ssh to an non-root user from host without using password







    - Set the rules of your firewall on your server only with the services used outside the VM.

    ⋆ in VirtualBox manager, your VM → settings → network, change network mode to NAT to be able to install, to Bridged Adapter to be able to ssh

    ⋆ set up UFW (Uncomplicated Firewall) to manage a netfilter firewall:


    ⋆ install UFW:    $ sudo apt install ufw


    ⋆ set default:     $ sudo ufw default deny incoming
    ⋆ set default:     $ sudo ufw default allow outgoing
    ⋆ allow HTTP:     $ sudo ufw allow 80/tcp
    ⋆ allow HTTPS:   $ sudo ufw allow 443
    ⋆ allow ssh port: $ sudo ufw allow [port]/tcp


    ⋆ enable:           $ sudo ufw enable
    ⋆ check status:  $ sudo ufw status verbose





    - Set a DOS (Denial Of Service Attack) protection on open ports of the VM.

    ⋆ set up Fail2Ban to secure the server against unauthorized access attempts:


    $ sudo apt-get install fail2ban


    $ sudo vim /etc/fail2ban/jail.d/jail-debian.local
    [DEFAULT]
    bantime = 10m
    findtime = 10m
    maxretry = 5

    [sshd]
    enabled = true
    port = [port]
    maxretry = 3
    findtime = 300
    bantime = 600
    logpath = %(sshd_log)s
    backend = %(sshd_backend)s

    [http-get-dos]
    enabled = true
    port = http,https
    filter = http-get-dos
    logpath = /var/log/apache2/access.log
    maxretry = 300
    findtime = 300
    bantime = 600
    action = iptables[name=HTTP, port=http, protocol=tcp]

    $ sudo vim /etc/fail2ban/filter.d/http-get-dos.conf
    [Definition]
    failregex = ^ -.*"(GET|POST).*
    ignoreregex =


    ⋆ apply settings:
    $ sudo ufw reload
    $ sudo service fail2ban restart
    ⋆ activate fail2ban:
    $ sudo systemctl enable fail2ban


    → to check Fail2ban status: sudo systemctl status fail2ban
    → to display Fail2ban rules: sudo iptables -L


    → to disable/enable UFW: sudo ufw disable/enable
    → to disable UFW & reset the rules: sudo ufw reset
    → to disable/enable Fail2ban: sudo service fail2ban stop/start


    - You have to set a protection against scans on your VM's open ports.

    set up PortSentry to detect and respond to port scans:


    $ sudo apt-get install portsentry


    ⋆ modify /etc/default/portsentry:
    TCP_MODE="atcp"
    UDP_MODE="audp"


    ⋆ modify /etc/portsentry/portsentry.conf:
    BLOCK_UDP="1"
    BLOCK_TCP="1"


    ⋆ comment all lines that start "KILL_ROUTE" except for:
    KILL_ROUTE="/sbin/iptables -I INPUT -s $TARGET$ -j DROP"


    ⋆ restart PortSentry:
    $ sudo /etc/init.d/portsentry start


    → PortSentry logs: sudo cat /var/log/syslog


    - Stop the services you don't need for this project.

    ⋆ display all services:
    $ ls /etc/init.d

    ⋆ display active services (are marked with [ + ]):
    $ sudo service --status-all

    ⋆ disable the unneeded ones:
    $ sudo systemctl disable bluetooth.service

    - Create a script that updates all the sources of package, then your packages and which logs the whole in a file named /var/log/update_script.log. Create a scheduled task for this script once a week at 4AM and every time the machine reboots.

    ⋆ create a script (in [user]/scripts/[updater_filename].sh):
    #!/bin/bash
    sudo apt-get update -y >> /var/log/update_script.log
    sudo apt-get upgrade -y >> /var/log/update_script.log


    ⋆ edit crontab to create a scheduled job:
    $ sudo crontab -e
    0 4 * * 0 sudo bash /home/[user]/scripts/[updater_filename].sh
    @reboot sudo bash /home/[user]/scripts/[updater_filename].sh
    $ sudo service cron restart





    - Make a script to monitor changes of the /etc/crontab file and sends an email to root if it has been modified. Create a scheduled script task every day at midnight.

    ⋆ create a script (in [user]/scripts/[cron_monitor_filename].sh):
    #!/bin/bash
    DIFF=$(diff /etc/crontab.back /etc/crontab)
    cat /etc/crontab > /etc/crontab.back
    if [ "$DIFF" != "" ]; then
        echo "crontab has been changed." | sudo mail -s "crontab alert" root@[host].[domain]
    fi

    ⋆ again add a scheduled job in crontab:
    0 0 * * * sudo bash /home/[user]/scripts/[cron_monitor_filename].sh

    WEB PART



    - Set a web server that is available on the VM's IP or a host. Set a self-signed SSL on all of your services.

    ⋆ copy your html document from host machine to ~/temp/index.html on the VM:
    $ scp -P [port] ~/[html-file] [user]@[ip]:~/temp/index.html
    ⋆ copy the html file in temp to index.html:
    $ sudo cp temp/index.html /var/www/html/index.html

    ⋆ install Apache2:
    $ sudo apt-get install apache2


    Create Self-Signed SSL Certifiacte for Apache in Debian 10:

    $ sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout /etc/ssl/private/apache-selfsigned.key -out /etc/ssl/certs/apache-selfsigned.crt

    Common Name: [ip]
    Email Address: root@[host].[domain]

    $ cat /etc/apache2/conf-available/ssl-params.conf

    SSLCipherSuite EECDH+AESGCM:EDH+AESGCM:AES256+EECDH:AES256+EDH
    SSLProtocol All -SSLv2 -SSLv3
    SSLHonorCipherOrder On

    Header always set X-Frame-Options DENY
    Header always set X-Content-Type-Options nosniff

    SSLCompression off
    SSLSessionTickets Off
    SSLUseStapling on
    SSLStaplingCache "shmcb:logs/stapling-cache(150000)"

    ⋆ in /etc/apache2/sites-available/default-ssl.conf:
    ServerAdmin root@localhost
    ServerName [ip]
    DocumentRoot /var/www/html
    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined
    SSLEngine on
    SSLCertificateFile /etc/ssl/certs/apache-selfsigned.crt
    SSLCertificateKeyFile /etc/ssl/private/apache-selfsigned.key


    ⋆ in /etc/apache2/sites-available/000-default.conf:
    Redirect "/" "https://[ip]/"

    $ sudo a2enmod ssl
    $ sudo a2enmod headers
    $ sudo a2ensite default-ssl
    $ sudo a2enconf ssl-params
    $ sudo apache2ctl configtest
    $ sudo systemctl restart apache2


    → to visit your page, type in the address bar of the browser: https://[ip]
    → tip: If Chrome does not show your site since it's not a trusted one, Firefox probably will

    DEPLOYMENT PART



    - A functional solution for deployment automation:

    ⋆ create a script in [user]/scripts/[html_updater_filename].sh that will compare temp/index.html to /var/www/html/index.html and update if changes have been made:
    #!/bin/bash
    DIFF=$(diff /home/[user]/temp/index.html /var/www/html/index.html)
    if [ "$DIFF" != "" ]; then
        sudo cp /home/[user]/temp/index.html /var/www/html/index.html
        echo "index.html has been changed." | sudo mail -s "html updated." root@[host].[domain]
    fi

    ⋆ add a cronjob to run updater script at desired times:
    * * * * * sudo bash /home/[user]/scripts/[html_updater_filename].sh

    ⋆ now you can just run this script to send the new version to vm once done editing your page, and the site will be automatically updated (* * * * * = every minute):
    #!/bin/bash
    scp -P [port] ~/index.html [user]@[ip]:~/temp/index.html
